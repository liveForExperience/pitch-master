# 领域模型 (Domain Model)

本文档定义了“老男孩俱乐部管理平台”的核心领域模型及其层级关系。所有代码实现、API 设计及数据库结构均应严格遵循此模型。

## 1. 领域层级 (Hierarchy)

| 层级 | 概念名称 | 业务定义 | 核心属性 |
| :--- | :--- | :--- | :--- |
| **L1: 租户层** | **Tournament (赛事)** | 系统的顶层容器，定义全局规则。 | 名称、规则配置、租户ID |
| **L2: 组织层** | **Club (俱乐部/球队)** | 球员的归属单位，属于具体赛事。 | 名称、Logo、所属赛事ID |
| **L3: 身份层** | **User / Player** | **User** 是账户主体（权限控制），**Player** 是竞赛主体（数据记录）。 | 账号、角色(admin/player)、能力值 |
| **L4: 活动层** | **Match (比赛)** | 一次具体的活动实例。 | 时间、报名列表、计划场次、状态机 |
| **L5: 执行层** | **Game (场次)** | Match 中的具体对阵。 | 比分、实时统计 |

## 2. 核心关系与角色

### 2.1 身份与权限 (Roles & Identity)
系统采用“身份与权限解耦”的设计，支持以下场景：
*   **Player (球员)**：系统中最基础的竞赛主体。在 `player` 表中有档案的用户即为球员。
*   **Admin (管理员)**：拥有赛事管理权限的用户。
*   **身份叠加 (Overlap)**：**一个用户可以同时拥有 `admin` 和 `player` 身份**。
    - 当其以 `admin` 身份操作时，可以发布赛事、处理结算。
    - 当其以 `player` 身份操作时，可以报名参赛、产生进球数据、参与评分。
*   **关系约束**：`User` 与 `Player` 始终保持 1:1 关系，确保每个账号对应唯一的竞技档案。

### 2.2 组织关系
* 一个 **Tournament** 包含多个 **Club**。
* 一个 **Player** 必须归属于一个 **Club**。
* **Admin** 属于 **Tournament**，拥有管理其下所有俱乐部、球员和比赛的权限。

### 2.3 活动与执行
* 一场 **Match** 包含多次 **Game**（基于分组算法动态生成）。
* **计划场次 (Planned Game Count)**：由管理员在发布活动时设定，直接影响系统自动排赛的数量。
* **Registration (报名)** 是球员（或未来俱乐部）参与 **Match** 的凭证。
* **Game Participant (场次参与者)** 是具体的表现记录：
    - 记录球员在特定 **Game** 中的进球、助攻、MVP 等数据。
    - 这是统计个人能力图表的最小数据单元。

## 5. 球员价值与评分体系 (Player Value & Rating)

系统建立了一套覆盖“单场表现”到“长期生涯”的动态评分模型，旨在实现公平分组并记录球员成长。

### 5.1 评分层级 (Layered Architecture)
| 层级 | 范围 | 定义 | 核心指标 |
| :--- | :--- | :--- | :--- |
| **原子层 (L1)** | Game | 单场对阵的原始表现。 | 进球、助攻、胜/平/负。 |
| **聚类层 (L2)** | Match | 针对一次完整活动的综合评估。 | 互评得分、MVP 荣誉、场均贡献。 |
| **全局层 (L3)** | Tournament | **核心分值**。代表球员在赛事中的数字化身。 | 综合评分 (0.0-10.0)。 |

### 5.2 核心算法规则
1.  **起评分 (Initialization)**：新用户创建档案时，默认分值为 **5.0** (区间 0.0 - 10.0)。
2.  **全自动演进 (Automatic Evolution)**：
    *   基于 **ELO 算法**：根据对手平均强弱计算胜负得分。
    *   **表现奖金**：进球 (+0.02)、助攻 (+0.01)。
    *   **荣誉加成 (MVP)**：一旦被确认为本场 MVP，获得显著额外加分 (**+0.10**)。
3.  **生命力与衰减 (Vitality & Decay)**：
    *   **活跃度权重**：系统引入时间衰减因子。
    *   **衰减逻辑**：若球员连续 4 周未参与任何 Match，其全局评分将开始缓慢衰减（建议每周 -0.05），直至回落至起评分 5.0。此举确保了“老本分”不会影响分组公平性。
4.  **互评反哺 (Social Feedback)**：
    *   球员互评中的“技术”与“意识”维度将以较低权重（如 10%-20%）定期校准全局层评分，以弥补纯数据计算的盲点。

### 5.3 评分的作用 (Utility)
*   **公平分组**：作为 `GroupingStrategy` 的首要加权因子。
*   **荣誉体系**：形成球员战力曲线和历史荣誉墙。
*   **准入门槛**：未来可用于设定高级别赛事的准入分值。
